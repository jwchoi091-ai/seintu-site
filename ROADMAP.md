# 수사해당가든 상업용 홈페이지 로드맵

> 작성일: 2026-06-25 · 최종 결정 반영본
> 이 문서는 사이트를 "소개 페이지"에서 "실제 판매·운영 가능한 상업 사이트"로 키우는 전체 계획입니다.

---

## 0. 확정된 전략 구조

소규모 농장이 회원·결제를 **직접** 구축하면 통신판매업 신고 외에도 PG사 계약·에스크로·개인정보 보안책임까지 떠안게 됩니다. 그래서 다음 구조로 확정했습니다.

- **결제·회원가입·정산·CS → 네이버 스마트스토어에 위임** (홈페이지에 결제/장바구니/회원DB를 직접 만들지 않음)
- **홈페이지 → 브랜드 + 규격별 카탈로그 + 견적 요청 깔때기** (R10·외목대·다간 같은 대형목은 정찰 결제가 아니라 견적형 상품)
- **재고·주문·견적 통합 운영 → 구글 시트 1개를 단일 원장으로** (전화·현장·스토어·홈페이지 주문을 한 표로 모으고 견적서를 자동 생성)

```
        홈페이지 (브랜드 + 카탈로그 + 견적 깔때기)
                │                       │
        소형·정찰제 상품            대형목·맞춤 주문
                │                       │
                ▼                       ▼
        네이버 스마트스토어          견적 / 문의 폼
        (결제·정산·CS 대행)        (전화·현장·온라인)
                │                       │
                └───────────┬───────────┘
                            ▼
        구글 시트 1개 (재고·주문·견적 단일 원장) → 견적서 자동 생성
```

---

## 1. 현재 상태 (출발점)

- React 18 + TypeScript + Vite 기반 **소개(메인) 페이지 1종**만 존재
- 백엔드·DB·로그인·결제·주문 기능은 아직 없음
- 모든 문구·메뉴·수치는 `src/data/site.ts` 한 곳에서 관리
- main 브랜치 push → Netlify 자동 배포 (1~2분)

---

## 2. 파트별 로드맵

### 파트 0 — 사업·법적 기반 (다른 파트와 병행 / 사장님 몫)
온라인 판매의 전제 조건. 코드보다 먼저 또는 같이 진행.
- [ ] 사업자등록 정보 확정 (상호·대표자·주소·사업자번호)
- [ ] 통신판매업 신고 (정부24, 구매안전서비스 가입 후)
- [ ] 개인정보처리방침 · 이용약관 · 청약철회/교환반품 정책 문안
- [ ] 카카오 채널 · 대표 연락처 · 정산 계좌 정리
- **산출물**: 홈페이지 푸터·스토어에 법적 표시할 정보 일체

### 파트 1 — 홈페이지 "완성" (디자인·페이지·모바일) ✅ 완료
- [x] 메인 페이지 디테일 다듬기 (히어로·통계·스토리 카피/이미지)
- [x] 상품 목록 페이지 + 상품 상세 페이지 (사진·규격·가격·식재정보)
- [x] 회사소개 / 농장 일정 / 앨범(갤러리) / 블로그·식재가이드 / 문의 페이지
- [x] 모바일 반응형 전 페이지 점검
- [x] 라우팅 도입 (react-router, 실제 페이지 분리 + Netlify SPA fallback)
- **산출물**: 클릭 가능한 다중 페이지 사이트, 모바일 정상 ✓

### 파트 2 — 상품/카탈로그 + 견적·문의 ✅ 기본 완료
- [x] 상품 데이터 구조 설계 (규격·재고수량·단가·사진·상태) — `site.ts` Product
- [x] 견적 요청 폼 (희망 규격/수량/식재지역/식재여부 → Netlify Forms로 접수)
- [x] 상품 상세 → 견적 폼 자동 연동 (상품명·규격 prefill)
- [x] 카카오톡 상담 연결 버튼 (채널 URL 설정 시 자동 노출)
- **산출물**: 고객이 대형목을 견적으로 문의, 제출은 Netlify Forms로 수집 ✓
- **사장님 할 일**: Netlify 대시보드 Forms → 알림 이메일 설정(1회), 카카오 채널·스토어 URL 입력

### 파트 3 — 회원 + 주문 (스마트스토어 위임)
- [ ] 홈페이지의 모든 "구매" 행동을 스토어로 연결 ("스토어에서 구매하기")
- [ ] 직접 회원·결제·DB 구축은 **하지 않음** (부담·비용·보안 회피)
- **산출물**: 결제는 스토어에서, 홈페이지는 깔끔한 깔때기

### 파트 4 — 네이버 스마트스토어
- [ ] 스토어 개설 + 사업자 인증 + 정산 계좌
- [ ] 소형·정찰제 상품(소형 묘목·원예자재) 우선 등록
- [ ] 홈페이지 ↔ 스토어 상호 링크, 브랜드 일관성
- **산출물**: 실제 결제 받는 채널 가동

### 파트 5 — 통합 운영 (재고·주문·견적 연동) ⭐ 착수
- [x] **운영 원장 v1 구축** — `운영원장/수사해당가든_운영원장.xlsx`
      (재고·거래·협력농가·교육·대시보드·견적서 7개 시트, 나무 특성 반영: 개체/로트, 실측 규격, 자가/협력농가 소유)
- [x] 견적서 양식 (품목 입력 → 합계 자동, 면세 반영)
- [x] 기간별(분기/반기/연) 매출·매입·마진 자동 집계
- [ ] 구글시트로 이관 + 계정 연동 (사장님 깨어 있을 때 1회)
- [ ] Netlify 견적 폼 제출 → 원장 자동 수집
- [ ] 재고 상태/교육 통계를 홈페이지에 자동 반영 (site.ts → 시트 읽기)
- **산출물**: 채널 무관 통합 주문·재고·견적 운영
- **비고**: 결제=스마트스토어, 계산서=홈택스(면세 여부 세무사 확인), 회계=추후 캐시노트/세무사

### 파트 6 — 마케팅·성장 (추천 기능)
- [ ] 네이버 검색 최적화 (서치어드바이저 등록)
- [ ] 네이버 플레이스(지도) 등록 → 농장은 지역검색이 매출 직결
- [ ] 식재 사례 갤러리 + 고객 후기 → 신뢰 = 구매전환
- [ ] 계절 예약 분양 (봄/가을 식재 시즌 사전예약·대기명단)
- [ ] SNS/블로그 연동, 간단 방문자·문의 분석(GA 등)

## 3. 실행 순서 요약

| 순서 | 파트 | 핵심 산출물 | 담당 |
|---|---|---|---|
| 1 | 파트 1 | 멀티 페이지 + 모바일 완성 (라우팅, 상품목록·상세, 갤러리, 문의) | 클로드 |
| 2 | 파트 2 | 상품 데이터 구조 + 견적 요청 폼 + 카톡 상담 | 클로드 |
| 3 | 파트 0 | 통신판매업 신고·사업자정보·약관 문안 | 사장님 (병행) |
| 4 | 파트 4 | 스마트스토어 개설 + 소형상품 등록 + 홈페이지 연결 | 사장님 주도, 클로드 지원 |
| 5 | 파트 5 | 구글 시트 원장 + 견적서 자동화 + 재고표시 연동 | 클로드 |
| 6 | 파트 6 | 네이버 플레이스·검색최적화·후기·계절예약 | 같이 |

> **다음 착수 지점**: 파트 1의 라우팅 도입 + 상품 목록/상세 페이지.
> 이게 있어야 견적 폼·스토어 연결·재고 표시가 모두 그 위에 얹힙니다.
