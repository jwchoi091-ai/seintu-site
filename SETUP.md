# 셋업 & 작업 안내 (수사해당가든)

이 문서 하나면 **다른 컴퓨터에서도** 이 프로젝트를 이어서 작업할 수 있습니다.

- 🌐 완성 사이트(라이브): https://sweet-lamington-f73d5c.netlify.app/
- 💾 코드 저장소(GitHub): https://github.com/jwchoi091-ai/seintu-site
- 이 프로젝트는 **수사해당화 전문 농장 '수사해당가든'**의 공식 홈페이지입니다. (현재: 소개 메인 페이지 완성)

---

## 1. 새 컴퓨터에서 처음 한 번만 (최초 세팅)

1. **Node.js LTS 설치** → https://nodejs.org
2. **Git 설치** → https://git-scm.com  (가능하면 GitHub CLI `gh`도: https://cli.github.com)
3. 터미널(명령창)에서 저장소 내려받기:
   ```bash
   git clone https://github.com/jwchoi091-ai/seintu-site.git
   cd seintu-site
   npm install
   npm run dev
   ```
   → 브라우저에서 `http://localhost:5173` 으로 사이트가 열립니다.

> 처음 `push`(올리기)를 할 때 GitHub 로그인이 1회 필요합니다. `gh auth login`을 하거나,
> push할 때 자동으로 뜨는 브라우저 인증 창에서 승인하면 됩니다.

---

## 2. 컴퓨터를 오갈 때 규칙 (가장 중요)

여러 컴퓨터에서 작업해도 **충돌 없이** 이어가려면 이 두 가지만 지키면 됩니다.

| 시점 | 할 일 | 의미 |
|---|---|---|
| **작업 시작 전** | `git pull` | 다른 컴퓨터에서 한 최신 내용 받아오기 |
| **작업 끝난 후** | `git add -A && git commit -m "내용" && git push` | 내 변경을 올리기 (→ 자동배포까지) |

흐름: `A컴퓨터에서 push` → `B컴퓨터에서 시작 전 pull` → 항상 최신 상태로 연결됨.

---

## 3. 자동배포 (이미 설정 완료)

`main` 브랜치에 **push 하면 Netlify가 자동으로 빌드·배포**합니다. (수동 업로드 불필요)

- 빌드 설정은 `netlify.toml`에 있음 (build: `npm run build`, publish: `dist`, Node 20)
- push 후 약 1~2분이면 라이브 사이트에 반영됨

---

## 4. Claude(클로드)에게 말로 시킬 때 — 추천 명령어

매번 명령어를 직접 칠 필요 없이, 클로드에게 아래처럼 말하면 대신 해줍니다.
**다른 요청과 헷갈리지 않도록 아래 표현을 권장**합니다.

| 하고 싶은 것 | 이렇게 말하기 (추천) | 클로드가 하는 일 |
|---|---|---|
| 작업 저장 + 사이트 반영 | **"저장하고 배포해줘"** | git add + commit + push → 자동배포 |
| 배포 없이 저장만 | **"커밋만 해줘"** | git add + commit (push 안 함) |
| 다른 PC의 최신 내용 받기 | **"깃허브에서 받아와줘"** | git pull |
| 지금 상태 확인 | **"깃 상태 보여줘"** | git status / 최근 커밋 |
| 이전 시점으로 되돌리기 | **"○○ 시점으로 되돌려줘"** | (신중히) 해당 커밋으로 복구 |
| 새 컴퓨터에서 시작 | **"이 저장소 세팅해줘"** | clone 안내 + npm install 등 |

### ⚠️ 피하면 좋은 모호한 표현
- "올려줘" → (스크롤/값 올리기/파일 업로드와 혼동) → 대신 **"저장하고 배포해줘"**
- "받아줘" → ('수락'과 혼동) → 대신 **"깃허브에서 받아와줘"**

## 5. 다른 컴퓨터의 클로드가 맥락을 모를 때

클로드의 프로젝트 기억(메모리)은 **컴퓨터마다 따로**입니다.
새 컴퓨터에서 처음 열면 이렇게 한마디 해주세요:

> "이 폴더는 수사해당가든 사이트야. README.md랑 SETUP.md 먼저 읽고 파악해줘."

그러면 코드 + 이 문서들을 보고 바로 이어서 작업할 수 있습니다.

---

## 6. 자주 쓰는 명령어 모음 (직접 칠 때)

```bash
npm run dev      # 개발 서버 실행 (작업 중 미리보기)
npm run build    # 배포용 빌드 (보통 자동배포가 대신 해줌)

git pull         # 최신 내용 받기 (작업 시작 전)
git status       # 현재 변경 상태 확인
git add -A       # 변경 전체 담기
git commit -m "메시지"   # 저장(커밋)
git push         # GitHub에 올리기 (→ 자동배포)
```
