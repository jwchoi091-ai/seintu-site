# 온라인 판매 개시 로드맵 — 수사해당가든

> **목적**: 홈페이지를 배포하고, 고객이 정상적으로 상품을 보고 → 결제 → 배송받기까지의 전 과정을 합법적으로 갖추기 위한 실행 보고서.
> 이 문서 하나를 보면서 순서대로 따라 작업하면 됩니다.
>
> - 작성일: 2026-06-30
> - 대상 상품: (현재) 수사해당화 **묘목 생산·판매** → (향후) 원예·농사 자재, 묘목 구매자 대상 **오프라인 교육·체험 예약**
> - 결제·배송 구조: **스마트스토어가 결제·배송·CS 담당**, 자사몰(현재 홈페이지)은 **브랜드 소개·상품 진열·유입** 담당

## 0. 한눈에 보는 전체 구조

```
   [고객]
     │  검색·SNS·지인 소개
     ▼
 ┌─────────────────────────┐        "구매하기" 클릭
 │  자사몰 (현재 홈페이지)   │ ───────────────────────┐
 │  netlify 배포            │   브랜드·소개·상품 진열   │
 │  src/data/site.ts        │                         ▼
 └─────────────────────────┘            ┌──────────────────────────┐
                                        │   네이버 스마트스토어      │
                                        │  결제 · 배송 · CS · 정산   │
                                        │  (구매안전서비스 포함)     │
                                        └──────────────────────────┘
                                                     │
                                              택배사 (CJ대한통운 등)
                                                     ▼
                                                  [고객 수령]
```

**핵심 판단**: 직접 PG(결제대행) 연동·자사몰 결제는 초기 비용·관리 부담이 큽니다.
묘목은 **규격·중량·배송난이도가 제각각**이라 표준 쇼핑몰 결제보다 스마트스토어의 옵션·배송비 설정 기능이 훨씬 유리합니다.
→ **자사몰 = 쇼윈도, 스마트스토어 = 계산대·창고**로 역할을 나누는 것이 가장 빠르고 저렴하며 안전합니다.

## 1. 진행 순서 (의존성 때문에 이 순서가 중요)

각 단계는 **앞 단계가 끝나야 다음이 가능**합니다. 특히 ②→③→④ 순서를 어기면 막힙니다.

| 순서 | 단계 | 어디서 | 소요 | 비용(대략) | 선행 조건 |
|---|---|---|---|---|---|
| ① | **사업자등록** | 홈택스 / 세무서 | 1일 | 무료 | 없음 |
| ② | **종자업(묘목) 등록** | 시·군 농업기술센터 / 국립종자원 | 3일~2주 | 등록면허세 약 4~5만원 + 시설 요건 | ① |
| ③ | **스마트스토어 개설** | 네이버 커머스 | 1~3일 | 무료 (수수료는 판매 시) | ① |
| ④ | **구매안전서비스 이용확인증 발급** | 스마트스토어 내 | 즉시 | 무료 | ③ |
| ⑤ | **통신판매업 신고** | 정부24 | 3~5일 | 등록면허세 약 4만원/년 | ④ (확인증 필요) |
| ⑥ | **상품 등록** (스마트스토어) | 스마트스토어 | 상품수에 따라 | 무료 | ③⑤ |
| ⑦ | **자사몰 ↔ 스마트스토어 연동** | 이 코드(홈페이지) | 1~2일 | 무료 (직접 작업) | ⑥ |
| ⑧ | **사업자 정보 푸터 표기 + 배포** | 이 코드 | 0.5일 | 무료 | ①⑤ |

> ⚠️ **반드시 기억할 의존성**: 통신판매업 신고(⑤)에는 **"구매안전서비스 이용확인증"**이 필요합니다.
> 이 확인증은 스마트스토어를 개설(③)하면 그 안에서 무료로 발급(④)됩니다.
> 그래서 **스마트스토어를 통신판매업 신고보다 먼저** 만들어야 합니다.

## 2. 단계별 상세

### ① 사업자등록 — *가장 먼저, 무료*

- **방법**: 홈택스(온라인) 또는 관할 세무서 방문. 온라인은 당일~익일 처리.
- **업종 선택(중요)**:
  - 묘목 판매 → **임업/원예 작물 재배·판매** 관련 업종코드
  - 통신판매를 할 것이므로 **"전자상거래 소매업"(업종코드 525101)** 을 함께 등록
  - 향후 교육·체험을 할 것이므로 여유 있으면 관련 업종도 추가 가능(나중에 추가도 됨)
- **과세 유형**:
  - 미가공 농산물·묘목은 **부가가치세 면세** 대상일 수 있으나, 원예자재 등 공산품을 함께 팔면 **과세**가 섞입니다. → **세무서/세무사에게 "묘목+자재+교육 혼합 판매" 라고 말하고 과세·면세 유형 확정**.
  - 농업인 자격이 있으면 농업경영체 등록 시 세제·정책자금 혜택이 있으니 별도 확인.
- **비용**: **무료**

### ② 종자업(묘목생산판매업) 등록 — *묘목 판매의 법적 전제*

- **근거**: 「종자산업법」. 묘목을 **생산·판매**하려면 종자업 등록이 필요할 수 있습니다(자가소비 수준 제외).
- **접수처**: 사업장 소재지 **시·군·구청 / 농업기술센터**, 일부는 **국립종자원**.
- **요건**: 일정 면적의 생산 시설·포장·검사 관련 요건. (이미 농장을 운영 중이면 충족이 쉬운 편)
- **비용**: 등록면허세 약 **4~5만원** 수준 + 지역별 차이. 시설 기준 충족이 핵심.
- **TODO**: 관할 농업기술센터에 전화해 **"수사해당화 묘목 판매하려는데 종자업 등록 대상인지, 요건이 무엇인지"** 먼저 확인 → 가장 불확실한 단계라 제일 먼저 문의 권장.

### ③ 스마트스토어 개설 — *결제·배송의 심장, 무료*

- **가입**: 네이버 커머스 → 사업자 판매자로 가입(①의 사업자등록증 필요).
- **비용**: **입점·개설 무료**. 비용은 **팔릴 때만** 수수료로 발생(§3 참조).
- **설정 포인트(묘목 특화)**:
  - 상품 옵션으로 **R규격(R3~R12)** 을 구성
  - **배송비**를 규격별로 차등(대형목은 화물·착불, 소형목은 택배)
  - 대형 성목은 **"구매 후 배송 협의"** 또는 **톡톡 문의형**으로 운영
- **결과물**: 스마트스토어 URL 1개 (예: `https://smartstore.naver.com/수사해당가든`)
  → 이 URL을 `src/data/site.ts`의 `business.smartStore`에 넣습니다.

### ④ 구매안전서비스 이용확인증 발급 — *통신판매업 신고용 서류*

- 스마트스토어 판매자센터 내 **"구매안전서비스 이용확인증"** 메뉴에서 즉시 PDF 발급(무료).
- 이 서류를 ⑤ 통신판매업 신고에 첨부합니다.

### ⑤ 통신판매업 신고 — *온라인 판매의 법적 자격*

- **접수처**: **정부24** 온라인 신고(또는 관할 시·군·구청).
- **첨부**: 사업자등록증 + ④ 구매안전서비스 이용확인증.
- **비용**: **등록면허세(매년 부과)**. 지역 인구 규모별 차등 —
  - 인구 50만 이상 시: 약 **40,500원/년**
  - 그 외 중소도시·군: 약 **22,500~40,500원/년** 구간
  - **첫 해 신고 시 1회 납부, 이후 매년 1월 자동 부과.**
- **결과물**: **통신판매업 신고번호** (예: `제2026-○○-0000호`)
  → `src/data/site.ts`의 `business.mailOrderNo`에 입력 → 홈페이지 푸터에 법적 표시(의무).

### ⑥ 상품 등록 (스마트스토어)

- 묘목 사진·규격·가격·재고·배송비를 스마트스토어에 등록.
- 자사몰의 `products` 더미 데이터(`src/data/site.ts`)와 **상품을 1:1 매칭**해 두면 ⑦ 연동이 쉬워집니다.
- **각 상품의 스마트스토어 상세페이지 URL**을 확보해야 ⑦에서 "구매하기" 버튼을 연결할 수 있습니다.

### ⑦ 자사몰 ↔ 스마트스토어 연동 *(개발 작업 — 내가 해드림)*

원하시는 동작: **자사몰 상품의 "구매하기" → 그 상품의 스마트스토어 구매 페이지로 이동** (모바일·웹 모두 자연스럽게).

구현 방식(권장안):
1. `Product` 타입에 `storeUrl?: string` 필드 추가 (각 상품의 스마트스토어 URL)
2. 상품 카드/상세에 **"스마트스토어에서 구매하기"** 버튼 → `storeUrl`로 새 탭 이동 (`target="_blank" rel="noopener"`)
3. `storeUrl`이 없는 상품(대형목·견적상품)은 기존처럼 **"견적·상담 문의"** 버튼 유지
4. 모바일에서는 버튼이 화면 폭에 꽉 차게, 데스크톱에서는 가격 옆에 배치 → 반응형 CSS로 자연스럽게

> 이 단계는 코드 작업이라, 스마트스토어 상품 URL만 확보되면 **"자사몰에 스마트스토어 구매 버튼 연결해줘"** 라고 말씀하시면 제가 처리합니다.

### ⑧ 사업자 정보 표기 + 최종 배포

- `src/data/site.ts`의 `business` 객체에 **실제 값** 입력:
  - `owner`(대표자명), `bizNo`(사업자등록번호), `mailOrderNo`(통신판매업 신고번호), `address`, `tel`, `smartStore`
- 전자상거래법상 **푸터에 상호·대표자·사업자번호·통신판매업 신고번호·주소·연락처 표시는 의무**입니다.
- 입력 후 **"저장하고 배포해줘"** → 라이브 반영.

## 4. 권장 일정 (타임라인 예시)

```
1주차  ① 사업자등록 (홈택스, 당일)
       ② 종자업 등록 — 농업기술센터 전화 문의 + 접수 (제일 먼저 문의!)
2주차  ③ 스마트스토어 개설 → ④ 구매안전확인증 발급
       ⑤ 통신판매업 신고 (정부24)
3주차  ⑥ 스마트스토어 상품 등록 (사진·규격·배송비)
       종자업 등록 결과 수령
4주차  ⑦ 자사몰 구매 버튼 연동 (코드 작업)
       ⑧ 사업자 정보 푸터 입력 → 저장하고 배포
       ▶ 정식 오픈
```

가장 변수가 큰 건 **② 종자업 등록**(지자체 처리 기간)이라, **1주차에 농업기술센터 전화부터** 하는 것이 전체 일정을 좌우합니다.

## 5. 향후 확장 (지금 안 해도 됨, 구조만 기억)
