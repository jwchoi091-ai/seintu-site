# Claude 사용 메모 (수사해당가든)

이 파일은 이 프로젝트에서 클로드(나)와 일할 때 기억해야 할 **명령어, 스킬, 워크플로우** 를 정리한 것입니다.
새로운 명령어나 스킬이 생기면 계속 업데이트됩니다.

---

## 1. 말로 시키는 명령어 (자연어)

다른 요청과 안 헷갈리게 아래 표현으로 말씀해 주세요.

| 하고 싶은 것 | 이렇게 말하기 | 내가 하는 일 |
|---|---|---|
| 작업 저장 + 사이트 반영 | **"저장하고 배포해줘"** | git add + commit + push → 자동배포 |
| 배포 없이 저장만 | **"커밋만 해줘"** | git add + commit (push 안 함) |
| 다른 PC 최신 내용 받기 | **"깃허브에서 받아와줘"** | git pull |
| 지금 상태 확인 | **"깃 상태 보여줘"** | git status / 최근 커밋 표시 |
| 이전 시점으로 되돌리기 | **"○○ 시점으로 되돌려줘"** | 해당 커밋으로 복구 (신중히) |
| 새 컴퓨터에서 시작 | **"이 저장소 세팅해줘"** | clone + npm install 안내 |

### ⚠️ 피하면 좋은 모호한 표현
- ~~"올려줘"~~ (스크롤/값 올리기 등과 혼동) → ✅ **"저장하고 배포해줘"**
- ~~"받아줘"~~ ('수락'과 혼동) → ✅ **"깃허브에서 받아와줘"**

## 3. 프로젝트 구조 (빠른 참고)

```
seintu-site/
├── src/
│   ├── components/       (UI 컴포넌트 + 스타일)
│   ├── pages/            (페이지)
│   ├── data/site.ts      ⭐ 모든 문구·메뉴·숫자의 단일 출처
│   └── styles/global.css (디자인 토큰)
├── index.html            (메타 태그, 라이브 버전)
├── SETUP.md              (다른 컴퓨터 세팅 안내)
├── README.md             (프로젝트 소개)
├── netlify.toml          (배포 설정 - 수정 금지)
└── CLAUDE.md             (이 파일)
```

### 콘텐츠 수정
- 모든 텍스트·메뉴·숫자 → `src/data/site.ts` 에서 수정
- 컴포넌트 추가/수정 → `src/components/` 의 `.tsx` + `.module.css`

---

## 4. 자동배포 (이미 설정됨)

- **트리거**: `main` 브랜치에 `git push`
- **빌드**: Netlify가 자동으로 `npm run build`
- **배포**: `dist/` 폴더가 라이브 사이트에 반영
- **소요 시간**: push 후 1~2분
- **라이브 사이트**: https://sweet-lamington-f73d5c.netlify.app/
- **GitHub 저장소**: https://github.com/jwchoi091-ai/seintu-site

---

## 5. 여러 컴퓨터에서 작업할 때

1. 새 컴퓨터: `git clone ...` + `npm install` (SETUP.md 참고)
2. 작업 시작 전: `git pull` (최신 내용 받기)
3. 작업 후: **"저장하고 배포해줘"** (push → 자동배포)
